# Nexus 设计愿景 (Design Vision)

> **版本**: 1.0 (Draft)
> **状态**: Draft
> **最后更新**: 2026-01-30

## 1. 项目背景
> *待补充：当前无线网络分析工具碎片化严重、数据可视化能力弱、OneDrive同步冲突等痛点分析。*

## 2. 设计哲学
### 2.1 物理分离，逻辑统一 (Federated Architecture)
Nexus 采用独特的联邦架构，将文档（Negentropy）与代码（Arsenal）在物理上分离以规避 OneDrive 同步问题，但在逻辑上通过 Agent 与工作区保持统一。

### 2.2 离岸平衡 (Offshore Balancing)
*   **Super Venv**: Python 环境全局化，避免污染项目目录。
*   **Local Node**: `node_modules` 仅存在于本地高性能存储中。

## 3. 架构全景
### 3.1 核心架构 (MVP Model)
*   **Model (后端)**: Python Flask + Pandas (数据处理)
*   **View (前端)**: React + Tailwind + NexusCharts (Plotly)
*   **Presenter (桥接)**: PyWebview (系统原生窗口)

### 3.2 数据流向
`Hardware/Driver -> Python Backend (Process) -> JSON -> React Frontend (Render)`

## 4. 演进路线 (Roadmap)
*   **v1.0 (Genesis)**: 基础框架跑通，Ping/Iperf 工具集成。
*   **v1.5 (Visualization)**: 引入科研级图表库 (NexusCharts)，重构调试界面。
*   **v2.0 (Intelligence)**: 引入 AI 辅助分析，自动化异常诊断。
