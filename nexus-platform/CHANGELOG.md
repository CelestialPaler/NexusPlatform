# 更新日志 (Changelog)

本项目的所有重要更改都将记录在此文件中。

## [1.5.0] - 2026-01-30

### 新增 (Added)
- **NexusCharts**: 引入基于Plotly的高性能科学图表库。
  - 支持 Line, Box, HeatMap, Bar, Area, Scatter, Pie, Histogram, Surface3D, Waterfall 等10种图表。
  - 实现全局 ThemeContext，图表自适应深色模式。
- **Debug Showcase**: 
  - 重构 Debug 页面为分栏布局 (Controls vs Charts)。
  - 增加可视化的图表案例展示区域。

### 变更 (Changed)
- **UI交互**: 将侧边栏 "Tools/工具" 重命名为 "Debug/调试"。
- **代码规范**: 全局统一前端代码的缩进与换行符 (CRLF/LF)。

## [1.4.1] - 2026-01-16

### 重构 (Refactor)
- **UI 组件标准化**: 完成前端 UI 的全面标准化重构。
  - **组件库升级**: 
    - 创建标准化的 `<Select>` 和 `<TextArea>` 组件。
    - 优化 `<Input>` 和 `<Select>` 组件，新增 `wrapperClassName` 属性以支持灵活布局。
  - **面板重构**: 全面替换原生 HTML 元素，统一使用 Common Components 实现所有交互控件。覆盖范围包括：
    - `AutomationPanel` (自动化面板)
    - `ScriptPanel` (脚本面板)
    - `ToolsPanel` (工具面板)
    - `ConditionBuilder` (条件构建器)
- **代码规范**: 移除了所有硬编码的 HTML 表单元素，统一了界面交互风格与底层实现。

## [1.4.0] - 2026-01-15

### 主要特性 (Feature)
- **工程化重构**: 完成了构建系统的全面分离，支持独立的 Release (发布) 与 Debug (调试) 构建流程。
- **独立运行**: 修复了 Release 版本脱离环境后的运行崩溃问题，确保了应用程序的独立部署能力。

### 变更 (Changed)
- **构建脚本分离**:
  - 新增 `scripts/build_release.ps1`: 生成无窗口、体积优化的发布版本。
  - 新增 `scripts/build_debug.ps1`: 生成带控制台、详细日志的调试版本。
  - 移除 `scripts/build_exe.ps1`: 废弃旧的单一构建脚本。

### 修复 (Fixed)
- **环境稳定性**: 修复了 release 版本下 `stdout` 引起的 IO 异常 (UnicodeEncodeError, Closed File Error)。
- **数据持久化**: 修复了自动化模块保存配置文件时丢失 `compositeScripts` 的严重 Bug。

## [1.3.0] - 2026-01-13

### 新增 (Added)
- **无线抓包与分析模块 (Wireless Capture & Analysis)**:
  - 新增无线网卡抓包工具，支持实时信道切换与数据包捕获。
  - 支持多网卡识别与模式切换 (Monitor/Managed)。
  - 集成 Pcap 文件保存与实时分析接口。
  - **可视化面板**: 提供独立的无线抓包控制面板，支持参数配置与状态监控。

### 优化 (Changed)
- **系统集成**: 将 Wireless Capture 模块集成至主应用框架。
- **依赖更新**: 新增 `pywin32` 和 `paramiko` 依赖支持。

## [1.2.0] - 2026-01-05

### 新增 (Added)
- **全功能自动化控制模块 (Automation Control Module)**:
  - **核心引擎**: 引入了全新的自动化执行引擎，支持模拟用户操作、任务调度与流程控制。
  - **可视化编排**:
    - **脚本管理**: 实现了基于文件夹的配置结构，支持多应用 (`profiles`) 和多脚本 (`scripts`) 的隔离与管理。
    - **条件构建器**: 新增可视化条件编辑器 (`ConditionBuilder`)，支持像素颜色匹配、图像识别等复杂逻辑判断。
    - **任务面板**: 提供直观的控制面板，支持脚本的录制、回放、调试及实时日志查看。
  - **高级特性**:
    - **周期性任务**: 支持配置定时或循环执行的任务流，满足长时间挂机或监控需求。
    - **多配置切换**: 支持不同应用场景（如"元梦之星"、"默认"）的快速切换。
- **Iperf 工具升级**: 增强了 Iperf 测试的稳定性，优化了参数配置与结果展示。

### 优化 (Changed)
- **架构重构**:
  - 重构了自动化模块的核心逻辑，提升了扩展性与维护性。
  - 升级了配置文件结构，从单文件迁移至文件夹模式，便于管理复杂配置。
- **文档更新**: 全面更新了开发指南 (`CODING_GUIDE.md`) 与架构文档，规范了代码风格与提交标准。

## [1.1.0] - 2025-12-24

### 新增 (Added)
- **多窗口支持**: 工具面板新增"独立窗口打开"功能，支持多任务并行运行。
- **关于页面**: 设置中新增关于信息模块，展示版本号、作者及联系方式。

### 优化 (Changed)
- **UI 简化**: 移除了 Ping 工具中冗余的 "New Task" 按钮。
- **稳定性修复**: 修复了 Advanced Ping 工具在独立窗口模式下因翻译资源加载导致的白屏问题。

## [1.0.1] - 2025-12-24

### 优化 (Changed)
- **视觉升级**: 全面重构所有工具 (RTP, Ping, Advanced Ping) 的 Logo，采用加粗 SVG 文本样式 (Extra Black)，解决了之前的图形变形问题。

## [1.0.0] - 2025-12-24

### 新增 (Added)
- **RTP 分析工具**:
  - 集成 RTP 协议分析功能，支持丢包率、抖动、帧率分析。
  - 支持自动检测 PCAP 文件中的 UDP 流。
  - 提供可视化图表展示帧大小和帧间隔。
  - 支持 RTSP 握手信令解析。
- **UI 标识**: 为基础 Ping (绿色) 和高级 Ping (橙色) 工具设计了区分度更高的 SVG 图标。

### 优化 (Changed)
- **工具分类**: 重组工具面板，划分为基础、高级和协议分析三类。
- **界面统一**: 统一了各工具面板的头部设计风格。

## [0.1.2] - 2025-12-23

### 修复 (Fixed)
- 修复了由高频 IPC 事件导致的 UI 卡死问题。
- 修复了后端的 "maximum recursion depth exceeded"（最大递归深度溢出）错误。
- 通过在生产运行中禁用调试模式，修复了 "Empty" 日志刷屏问题。

### 变更 (Changed)
- 在 `WindowManager` 中实现了事件批处理，以优化前端更新。
- 将 Python 递归限制增加到 2000。
- 将默认日志级别更改为 INFO。
- 优化了 `PingManager` 和 `IperfManager` 以使用批量事件发送。
